## Tasks

### T-01: Complete Domain Model

**File:** `inbox/models.py`

Implement full SQLModel entities:

```python
class Item(SQLModel, table=True):
    id: UUID = Field(default_factory=uuid4, primary_key=True)
    type: str = Field(index=True)  # text | url | image | file
    captured_at: datetime = Field(default_factory=utc_now)
    updated_at: datetime = Field(default_factory=utc_now)
    raw_text: str | None = None
    blob_path: str | None = None
    metadata: dict = Field(default_factory=dict, sa_column=Column(JSONB))
    enrichment: dict = Field(default_factory=dict, sa_column=Column(JSONB))
    search_vector: str | None = Field(default=None, sa_column=Column(TSVECTOR))
    embedding: list[float] | None = Field(default=None, sa_column=Column(Vector(384)))
    deleted_at: datetime | None = None

    tags: list[Tag] = Relationship(back_populates="items", link_model=ItemTag)
    categories: list[Category] = Relationship(...)
```

Add Pydantic validators:
- Auto-detect `type` from `raw_text` (URL regex, image mimetype, file extension)
- Validate `metadata` structure based on `type`
- Ensure `raw_text` or `blob_path` is present

### T-02: API Schemas

**File:** `inbox/server/schemas.py`

Create Pydantic schemas for:
- `ItemCreate` — fields required for creation
- `ItemUpdate` — fields allowed for partial update
- `ItemResponse` — full item with tags and categories
- `SearchRequest` — search query + filters
- `SearchResponse` — ranked results
- `TagCreate`, `TagResponse`
- `CategoryCreate`, `CategoryResponse`
- `StatsResponse`

### T-03: API Routers

**Files:** `inbox/server/routers/*.py`

Implement all endpoints:

**Items Router:**
- `POST /v1/items` — Create item with auto type detection
- `GET /v1/items` — List with pagination (limit/offset), type filter, deleted filter
- `GET /v1/items/{id}` — Get by ID (404 if soft-deleted)
- `PATCH /v1/items/{id}` — Partial update
- `DELETE /v1/items/{id}` — Soft delete

**Search Router:**
- `GET /v1/search?q=...&type=...` — Full-text search
- `POST /v1/search/similar` — Vector similarity search (body: embedding or item_id)

**Tags Router:**
- `GET /v1/tags` — List all tags
- `POST /v1/tags` — Create tag
- `GET /v1/tags/{id}` — Get tag with items
- `DELETE /v1/tags/{id}` — Delete tag

**Categories Router:**
- `GET /v1/categories` — List all categories
- `POST /v1/categories` — Create category
- `GET /v1/categories/{id}` — Get category with items

**Stats Router:**
- `GET /v1/stats` — Total items, by type, by tag, storage usage

### T-04: API Client for CLI

**File:** `inbox/cli/client.py`

Create reusable httpx client:
```python
class InboxClient:
    def __init__(self, base_url: str = "http://localhost:8080"):
        self.client = httpx.Client(base_url=base_url, timeout=30.0)

    def create_item(self, raw_text: str, type: str | None = None) -> dict: ...
    def list_items(self, limit: int = 20, offset: int = 0) -> list[dict]: ...
    def get_item(self, item_id: str) -> dict: ...
    def update_item(self, item_id: str, **fields) -> dict: ...
    def delete_item(self, item_id: str) -> None: ...
    def search(self, query: str) -> list[dict]: ...
```

Refactor CLI commands to use `InboxClient`.
